# 写作工坊（Writing Studio）设计

## 目标

写作工坊帮助用户检查自己写下的英文短句或文章，保留原文、最小纠错版本、自然表达版本和 AI 解释。用户可选择推荐词汇加入现有词汇库，也可把一批历史记录整理成独立的阶段总结，再永久删除原始内容。

首版稳定支持约 300 英文词的作文。短句与作文共用录入流程，但作文额外提供全文优化、结构、连贯性和语气反馈。参考范文由用户选择是否生成。

## 产品入口

写作工坊有两个入口：

1. 现有悬浮工具圆标中的“写作纠错”，用于快速录入、分析和确认保存。
2. 侧边栏独立“写作工坊”页面，用于浏览记录、搜索、批量总结和删除。

快速入口与独立页面使用同一套分析预览组件，避免两套行为分叉。

## 核心流程

```text
输入短句或文章
      |
AI 自动判断类型和文体
      |
生成全文反馈与逐句分析
      |
用户预览、编辑、选择词汇
      |
确认后写入数据库
      |
浏览、搜索、多选记录
      |
生成阶段总结
      |
可永久删除源记录
      |
多选阶段总结，再次汇总
```

分析提交前只存在于前端草稿。只有用户点击“确认保存”才写入数据库。关闭未保存结果时必须二次确认。

## AI 输出

每个句子包含：

- `original`：用户原句。
- `corrected`：只修正语法、拼写和明显搭配错误，尽量少改。
- `natural`：保持原意的自然英文表达。
- `explanation`：AI 自由文本解释，不建立固定语法标签。
- `vocabulary`：适合当前语境的推荐词汇。

每个推荐词汇包含：

- 推荐单词或表达。
- 中文含义。
- 推荐原因。
- 使用该词重写当前原句的例句。

作文额外包含：

- 自动识别的写作类型或文体。
- 总体评价。
- 保持原意和事实的优化后全文。
- 结构与逻辑建议。
- 连贯性建议。
- 语气与自然度建议。
- 逐句分析。

AI 不得静默删除信息或虚构事实。优化后全文必须与逐句自然表达保持语义一致。

## 可选参考范文

“生成范文”默认关闭。用户可在分析前选择生成一至两篇，也可在分析完成后单独触发，无需重新分析全文。

优化后全文基于用户原文修改；参考范文围绕相同主题重新创作，可采用不同结构或表达角度。界面必须明确区分两者。

范文不参与用户错误统计。范文可复制、保存；其中表达可由用户单独加入现有词汇库。

## 数据模型

### `writing_submissions`

- `id`
- `original_text`
- `input_type`：`sentence` 或 `essay`
- `detected_genre`
- `overall_feedback`
- `refined_full_text`
- `structure_feedback`
- `coherence_feedback`
- `tone_feedback`
- `sentence_count`
- `created_at`

### `writing_sentences`

- `id`
- `submission_id`
- `position`
- `original`
- `corrected`
- `natural`
- `explanation`

### `writing_vocabulary`

- `id`
- `sentence_id`
- `original_expression`
- `suggested_word`
- `meaning`
- `reason`
- `example_sentence`
- `selected`
- `vocabulary_id`：可空，指向现有词汇库记录

推荐词在写作工坊中始终可见。用户选择保存后，词汇同时进入现有词汇库，来源标记为 `writing`，例句使用 `example_sentence`。写作原记录之后被删除时，已加入现有词汇库的词及例句继续保留。

### `writing_model_essays`

- `id`
- `submission_id`
- `position`
- `content`
- `created_at`

### `writing_summaries`

- `id`
- `title`
- `content`
- `source_type`：`sentences`、`submissions` 或 `summaries`
- `source_snapshot`
- `created_at`

总结必须保存自包含快照，不在渲染时依赖源记录。多份总结可作为新总结的输入。

## 写作工坊页面

页面包含“写作记录”和“学习总结”两个页签。

### 视觉与布局

页面采用“现代写作编辑器 + 高效开发工具”的组合风格。顶部展示 Writing Studio 标题、简短说明、AI Provider 状态，以及“新建写作、写作记录、学习总结”三个主视图入口。

新建写作在桌面窗口使用左右分栏：左侧为大尺寸、低干扰原文编辑器，底部显示词数、范文选项和分析按钮；右侧承载总体评价、逐句 Corrected、Natural、解释和推荐词。分析结果出现前，右侧显示有用途说明的空状态。窄窗口切换为单栏。

写作记录采用主从布局：左侧为紧凑记录列表，展示日期、类型、原文预览和句数；右侧阅读选中记录。原文、Corrected 和 Natural 使用清晰的对比样式。推荐词独立展示含义、原因和完整语境例句。搜索与批量操作集中在顶部工具栏，不把所有记录纵向展开为大型卡片。

学习总结采用左侧时间线、右侧 Markdown 阅读区。“生成文档”是阅读区主操作；汇总和永久删除属于次级批量操作。Markdown 必须渲染标题、粗体、引用和列表，不直接显示 `#`、`**` 等源标记。

全页充分使用桌面空间，不设置过窄的内容最大宽度。减少装饰性边框，使用背景层级、间距和排版分组；主色只用于当前状态与主要操作。正文保持舒适字号和行高，辅助元数据降低视觉权重。空状态必须提供明确下一步入口。

写作记录支持：

- 按一次提交展开多句。
- 搜索原句、纠正版、自然版、AI 解释和推荐词。
- 选择单句或整次提交。
- 编辑保存后的文本字段。
- 纵向阅读视图和原句、纠错、自然表达三栏对比视图。
- 点击已入库推荐词打开现有词汇详情。
- 单条或批量永久删除。
- 对所选内容生成阶段总结。

学习总结支持：

- 查看和编辑总结标题、正文。
- 多选历史总结再次汇总。
- 单条或批量永久删除。

## 阶段总结

阶段总结只使用用户所选句子、作文或历史总结，不隐式扫描整个数据库。建议输出：

- 整体表现。
- 高频语法问题。
- 不自然表达模式。
- 词汇使用建议。
- 代表性的原句与改进句。
- 下一阶段练习建议。

生成完成后，界面询问是否永久删除本次使用的源记录。新总结默认不删除旧总结。

### 生成文档

每份学习总结提供“生成文档”操作。点击后显示确认 Modal，允许编辑文档标题并选择两个目标之一：

1. **TanWords 数据库**：创建正式数据库文档，标题使用 Modal 中的标题，正文使用总结 Markdown。保存成功后提供前往“文档”页面的操作。
2. **选择本地目录**：确认后打开系统目录选择器，在目标目录创建 Markdown 文件。文件名由标题生成，过滤操作系统不允许的字符；同名时自动添加序号，不覆盖已有文件。保存成功后显示完整路径。

导出的 Markdown 格式为：

```md
# 2026-07-18 写作总结

> 生成于 2026-07-18 21:26

总结正文……
```

导出失败时保留 Modal 内容并允许重试。生成文档不修改或删除原学习总结。不提供“当前已挂载文档目录”作为独立目标。

## 删除规则

删除为永久删除，不提供归档或回收站。

- 删除 Submission 时级联删除其句子、AI 分析、范文和写作工坊内的推荐词记录。
- 已复制进现有词汇库的词和例句不删除。
- 总结快照不受源记录删除影响。
- 删除确认框显示将删除的提交、句子、分析和推荐词数量，并明确显示不会删除的词汇库记录数量。

## AI 数据流

AI 使用现有 Provider 和密钥存储。分析结果采用结构化响应：

```ts
interface WritingAnalysis {
  detectedType: "sentence" | "essay";
  detectedGenre: string;
  overallFeedback: string;
  refinedFullText: string;
  structureFeedback: string;
  coherenceFeedback: string;
  toneFeedback: string;
  sentences: Array<{
    original: string;
    corrected: string;
    natural: string;
    explanation: string;
    vocabulary: Array<{
      word: string;
      meaning: string;
      reason: string;
      exampleSentence: string;
    }>;
  }>;
}
```

前端流式展示进度，但只有完整结构通过校验后才允许确认保存。保存使用数据库事务，禁止产生半条 Submission。

约 300 词作文使用单次分析请求。更长内容可按段分析，再生成全文反馈；首版不承诺超长文档性能。范文使用独立请求。

## 错误处理

- 空输入或没有英文内容时不发送请求。
- 网络中断、响应截断或结构化解析失败时保留原输入，允许重试。
- AI 缺少必要字段时显示明确错误，不写数据库。
- 每个返回句子必须能对应原文；不匹配时要求重新分析。
- 关闭包含未保存分析的面板时要求确认。
- DB transaction 失败时整体回滚并保留前端预览。

## 测试范围

- 单句、多句和约 300 词作文分析。
- 自动区分短句与作文、自动识别文体。
- 最小纠错与自然表达语义一致性。
- AI 缺字段、截断 JSON、网络中断和重试。
- 保存前编辑分析结果。
- 推荐词例句生成、选择入库及现有词汇关联。
- 删除写作记录后，已入库词汇和例句仍存在。
- 源记录删除后，总结快照仍完整。
- 选择句子、整篇作文生成总结。
- 多份总结再次汇总。
- 可选生成一篇或两篇范文。
- 中英文混合输入和空输入。
- 批量永久删除的计数、确认与级联行为。

## 不在首版范围

- 固定语法分类、错误次数统计和规则化标签系统。
- 自动保存未确认的 AI 分析。
- 云同步或多人协作。
- 归档、回收站和恢复删除记录。
- 对超长书籍或论文提供完整文档分析保证。
